# Login And Access Control OKR Example

## 甲方需求

做一个用户登录与权限管理模块，支持注册、登录、退出和角色权限控制。

## Boss OKR

O-BOSS：交付一个安全、可用、可扩展的登录与权限管理能力。

| KR | 验收标准 | 证据要求 | 状态 |
| --- | --- | --- | --- |
| B-KR1 | 用户可完成注册、登录、退出 | 接口、页面、测试记录 | 进行中 |
| B-KR2 | 支持角色权限控制 | 权限矩阵、权限测试 | 未开始 |
| B-KR3 | 关键流程有安全校验 | 安全检查清单 | 未开始 |

## 角色树

```text
Boss
├─ 产品经理
└─ 架构师
   ├─ 后端开发
   ├─ 前端开发
   └─ 测试工程师
```

## 层级 OKR

### 产品经理

上级映射：B-KR1, B-KR2

O：明确登录与权限模块的产品范围和验收标准。

- PM-KR1：输出注册、登录、退出 3 条核心流程。
- PM-KR2：输出角色权限矩阵。

### 架构师

上级映射：B-KR1, B-KR2, B-KR3

O：形成可实现、可测试的登录权限技术方案。

- ARCH-KR1：输出接口、数据模型和权限模型。
- ARCH-KR2：拆解后端、前端、测试 KR。
