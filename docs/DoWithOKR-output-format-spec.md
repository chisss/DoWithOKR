# DoWithOKR 输出格式规范

**版本**：0.1  
**日期**：2026-04-30  
**用途**：规定各技能的标准展示格式，让用户稳定看到 OKR 树、交付幕进度、状态看板、对齐检查和评分复盘。

## 1. 通用展示原则

- 先展示结论，再展示明细。
- 所有 KR 必须显示上级 KR 映射。
- 所有状态必须使用：未开始、进行中、阻塞、已完成、放弃。
- 所有完成声明必须绑定证据。
- 所有评分必须说明评分人、被评分人、证据和差距。
- 默认输出摘要；用户要求“展开”时再输出完整明细。

## 2. `okr-boss` 输出

```markdown
## Boss OKR

甲方需求：做一个用户登录与权限管理模块

O-BOSS：交付一个安全、可用、可扩展的登录与权限管理能力

| KR | 验收标准 | 证据要求 | 状态 |
| --- | --- | --- | --- |
| B-KR1 | 用户可完成注册、登录、退出 | 接口/页面/测试记录 | 未开始 |
| B-KR2 | 支持角色权限控制 | 权限矩阵、测试用例 | 未开始 |
| B-KR3 | 关键流程有安全校验 | 安全检查清单 | 未开始 |

边界：不包含第三方 OAuth，不包含企业 SSO。
待确认：是否需要短信验证码？
```
