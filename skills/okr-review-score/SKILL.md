# OKR Review Score

## 执行规则

- 用户可主动触发评分复盘。
- 评分采用上级给直接下级打分。
- 评分必须绑定证据和差距说明。
- 每个角色的 Objective 得分默认取 KR 平均分。
- Boss 最终 R 由产品线 R、技术线 R、证据质量和甲方验收结论共同决定。

## 输出格式

使用 `references/score-review-template.md`。

必须包含：

- OKR 评分复盘
- 上级评分
- 角色 R
- Boss 最终 R
- 下一轮建议
