# OKR Execution Plan

## 执行规则

- 基于已确认的 Boss OKR、角色树和角色 OKR 生成计划。
- 计划必须按交付幕组织，不用现实时间周期承诺。
- 每个任务必须标注负责人、上级映射、证据和退出门禁。
- 状态只能使用：未开始、进行中、阻塞、已完成、放弃。
- 发现无人负责的 Boss KR 时必须阻塞并提示。

## 输出格式

输出交付幕计划表，字段包括幕、目标、负责人、任务、上级映射、证据要求、退出门禁。

必须包含：

- 交付幕计划
- 上级映射
- 证据
- 退出门禁
