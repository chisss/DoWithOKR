# 证据管理规范

本文档定义 DoWithOKR 插件的证据管理机制，包括证据类型、存储约定、文件格式、自动收集规则和验证规则。

## 证据类型

| 类型 | 标识 | 说明 | 示例 |
| --- | --- | --- | --- |
| 文件路径 | `文件` | 执行过程中创建或修改的源码/配置文件 | `src/api/login.ts` |
| Git Commit | `commit` | 关联的 git commit hash | `a1b2c3d` |
| 测试结果 | `测试` | 单元测试、集成测试、E2E 测试的执行结果 | `tests/login.spec.ts 通过` |
| 截图 | `截图` | UI 截图、运行效果截图 | `docs/screenshots/login.png` |
| 日志 | `日志` | 构建日志、部署日志、运行日志 | `logs/build-2026-04-30.log` |
| 文档 | `文档` | 设计文档、API 文档、用户手册等 | `docs/api-spec.md` |

## 存储约定

- 路径：`.okr/evidence/{KR-ID}.md`，每个 KR 一个证据文件。
- 文件名使用 KR 编号，如 `GM-KR1.md`、`PD-KR1.md`、`PM-KR2.md`、`BE-KR1.md`。
- 文件不存在时由 `okr-role-run` 自动创建。
- 证据文件随执行追加，不覆盖历史条目。

## 证据文件格式

```markdown
# {KR-ID} 证据

| 类型 | 路径/引用 | 说明 | 时间 |
| --- | --- | --- | --- |
| 文件 | src/api/login.ts | 登录接口实现 | 2026-04-30 |
| 测试 | tests/login.spec.ts | 登录单元测试通过 | 2026-04-30 |
| commit | a1b2c3d | feat: 实现登录接口 | 2026-04-30 |
```

- 表头固定为 `| 类型 | 路径/引用 | 说明 | 时间 |`。
- 类型字段必须使用上述枚举值之一。
- 时间格式：`YYYY-MM-DD`。
- 每次执行追加新行，保留历史记录。

## 自动收集规则

`okr-role-run` 执行完成后，自动扫描并收集候选证据：

1. **文件变更扫描**：对比执行前后的 `git diff --name-only`，将新增/修改的文件记录为 `文件` 类型证据。
2. **Git Commit 关联**：如果执行过程中产生了 commit，记录 commit hash 为 `commit` 类型证据。
3. **测试结果捕获**：如果执行了测试命令，将测试通过/失败结果记录为 `测试` 类型证据。
4. **日志与截图**：执行过程中产生的日志文件或截图，记录为对应类型证据。
5. **文档产出**：PD 产品总监、PM 产品经理、TW 文档专家等角色产出的文档，记录为 `文档` 类型证据。

## 证据验证规则

评分复盘（`okr-review-score`）时，按以下规则验证证据有效性：

| 条件 | 判定 | 处理 |
| --- | --- | --- |
| KR 状态为"已完成"且有 ≥1 条有效证据 | 有效 | 正常评分 |
| KR 状态为"已完成"但无证据 | 无效 | 标记为"待验证"，建议补充证据 |
| 证据路径指向不存在的文件 | 可疑 | 降低可信度，提示验证 |
| 证据类型与角色职责不匹配 | 弱证据 | 可接受但评分时权重降低 |
| 有 commit 证据且可通过 git log 验证 | 强证据 | 评分时权重提升 |

## 证据链接

- `status.md` 的证据列引用格式：`[N 条](.okr/evidence/{KR-ID}.md)`，其中 N 为证据条目数。
- 无证据时显示 `无`。
- `okr-review-score` 评分时从证据文件读取完整清单，交叉验证 KR 验收标准。
