# GM OKR Template

## 甲方需求

用一句话复述用户的自然语言需求，并标记已确认与待确认内容。

- **已确认**：用户明确表达的功能、范围、约束
- **待确认**：模糊表述、隐含假设、未提及但可能影响交付的内容

## GM Objective

以甲方视角表达本次需求要达成的业务或产品结果。Objective 应当：
- 方向明确，一句话概括最终交付价值
- 不包含具体技术方案或实现细节
- 可被非技术人员理解

## KR 万能转化公式

```
KR = 时间节点 + 动作交付物 + 质量/效率指标
```

此公式解决 90% 技术执行岗写 OKR 的痛点。KR 不是"做了什么"，而是"在什么时间前，交付了什么，达到了什么标准"。

### 转化对照表

| 日常技术动作 | ❌ 错误 KR（写动作） | ✅ 正确 KR（写交付标准） |
|------------|-------------------|----------------------|
| 开发功能模块 | 开发用户管理模块 | M3 结束前完成用户模块开发，单测覆盖率 ≥ 90% |
| 改 bug | 修复线上 bug | 本周线上 bug 24h 响应，修复率 ≥ 95% |
| 写文档 | 写接口文档 | M3 结束前完成接口文档，联调零阻塞 |
| 优化性能 | 优化检索速度 | M3 结束前完成检索优化，响应时间降低 30% |
| 代码审查 | 做 code review | 本周 PR 审查覆盖率 100%，平均审查时间 < 4h |
| 安全加固 | 做安全扫描 | M3 结束前完成 OWASP Top10 扫描，高危漏洞归零 |

### KR 质量检查清单

- [ ] **Specific（具体）**：KR 描述了一个明确的可交付物或可观测的结果
- [ ] **Measurable（可衡量）**：有清晰的验收标准，能判断"做到了"还是"没做到"
- [ ] **Achievable（可达成）**：在当前资源和时间约束下可以完成
- [ ] **Relevant（相关）**：直接支撑 GM Objective，删掉它会影响目标达成
- [ ] **Time-bound（有时限）**：有明确的时间节点（交付幕或具体日期），不是"未来某天"
- [ ] **Deliverable（交付标准）**：KR 包含时间节点 + 交付物 + 至少一个量化指标

### KR 反模式检测规则

生成 KR 后，必须逐条检测以下反模式。命中任一条即为不合格，必须修正后才能进入下一阶段。

| 反模式 | 检测规则 | 修正方式 |
| --- | --- | --- |
| 纯动作描述 | 包含"开发"、"实现"、"编写"、"搭建"、"优化"等动作词，但无量化质量指标 | 补充量化指标：覆盖率、响应时间、完成率、错误率等 |
| 无时间节点 | KR 中无交付幕（M2/M3）或具体日期 | 补充时间节点 |
| 不可验证 | 第三方无法独立判断是否达标（如"代码质量良好"） | 改为可量化的验收条件 |
| KR 太模糊 | 无法明确指出"交付了什么"（如"提升用户体验"） | 改为具体交付物 + 量化指标 |
| KR 是任务 | 描述的是活动而非结果（如"编写单元测试"） | 改为结果："核心模块测试覆盖率 ≥ 80%" |
| KR 与 O 脱节 | KR 完成了但 Objective 没有推进 | 重新审视 KR 是否真正支撑 Objective |
| KR 超出周期 | 单个 KR 需要多个迭代才能完成 | 拆分为当前周期可交付的子结果 |

## 边界

列出本轮明确不做的内容，避免范围蔓延。

### 边界判断规则

- **纳入**：用户明确要求 + 直接影响 Objective 达成的内容
- **排除**：用户未提及且不影响核心交付的内容；"锦上添花"类优化
- **待定**：用户提及但描述模糊，需确认后再决定纳入或排除

## 待确认

列出需要用户确认的问题；确认前不得把不确定内容标记为已完成。

## 示例：一个合格的 GM OKR

**甲方需求**：为内部管理系统增加用户权限模块，支持角色分配和菜单控制。

**GM Objective**：交付可用的 RBAC 权限管理模块，使管理员能按角色控制用户的菜单和操作权限。

| KR ID | 时间节点 | 交付物 | 质量指标 | 证据要求 | 状态 |
| --- | --- | --- | --- | --- | --- |
| GM-KR1 | M3 结束前 | 角色权限分配功能上线 | 管理员可创建角色并分配菜单权限，保存后 5s 内生效 | 操作录屏 + 数据库记录截图 | 未开始 |
| GM-KR2 | M3 结束前 | 菜单权限隔离功能上线 | 用户登录后仅看到其角色对应的菜单项，越权访问返回 403 | 不同角色登录对比截图 + 接口测试报告 | 未开始 |
| GM-KR3 | M3 结束前 | 权限热生效能力 | 权限变更不需要重启服务，热生效延迟 < 5s | 变更前后对比日志 + 时间戳截图 | 未开始 |
