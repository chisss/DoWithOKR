# Score Review Template

## OKR 评分复盘

- **复盘范围**：本次交付周期内所有角色的 KR
- **评分口径**：上级对直接下级逐条 KR 打分，分值 0.0 ~ 1.0
- **证据来源**：状态看板中记录的证据链接

## 评分标准

| 分数 | 含义 | 判定条件 |
| --- | --- | --- |
| 0.0 | 未开始/无证据 | KR 未执行，或无任何可验证的产出 |
| 0.3 | 有进展但未达标 | 有部分产出，但未满足验收标准的核心条件 |
| 0.7 | 基本达标 | 验收标准的核心条件已满足，存在次要缺陷或边界未覆盖 |
| 1.0 | 完全达标 | 验收标准全部满足，证据充分，无遗留问题 |

### 评分原则

- 只认证据，不认"我觉得做了"
- 0.3 和 0.7 之间不设中间值，强制二选一以减少模糊地带
- 对于 `放弃` 状态的 KR，不参与评分，从分母中剔除

## 分数汇总公式

### 产品线 R（Product Line Result）

```
产品线 R = PD 及其下级（PM, UI, TW）所有 KR 分数的算术平均
```

### 技术线 R（Tech Line Result）

```
技术线 R = ArchD 及其下级（BE, FE, QA, DevOps, SEC）所有 KR 分数的算术平均
```

### GM 最终 R

```
GM 最终 R = 产品线 R × 40% + 技术线 R × 60%
```

## 下一轮建议

### 基于分数的建议逻辑

| GM 最终 R | 建议 |
| --- | --- |
| ≥ 0.8 | 目标基本达成，下一轮可提升挑战度或扩展范围 |
| 0.5 ~ 0.7 | 部分达标，下一轮聚焦未完成 KR，优先补齐短板 |
| < 0.5 | 目标严重偏离，需复盘根因：范围过大？资源不足？方向错误？ |
