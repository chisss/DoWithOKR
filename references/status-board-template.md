## 状态定义与流转规则

合法状态值：`未开始`、`进行中`、`已完成`、`阻塞`、`放弃`

```text
未开始 → 进行中    （开始执行时）
进行中 → 已完成    （证据齐全且验收通过）
进行中 → 阻塞      （遇到外部依赖或不可控因素）
阻塞   → 进行中    （阻塞解除后）
进行中 → 放弃      （经 GM 确认，该 KR 不再需要）
未开始 → 放弃      （经 GM 确认，该 KR 不再需要）
```

### 流转约束

- `已完成` 和 `放弃` 是终态，不可回退
- 标记 `已完成` 前必须填写 `证据` 列，无证据不得标记完成
- 标记 `阻塞` 时必须填写 `阻塞原因` 列
- 标记 `放弃` 必须经过 GM 确认并记录原因

## 进展计算公式

单个 KR 进展值范围：`0.0` ~ `1.0`

### 角色级进展

```
角色进展 = 该角色所有 KR 进展值的算术平均
```

### 幕级进展

```
幕进展 = 该幕所有 KR 进展值的算术平均
```

### 总体进展

```
总体进展 = 所有非放弃 KR 进展值的算术平均
```

注意：状态为 `放弃` 的 KR 不参与进展计算。

## 阻塞升级规则

| 阻塞时长 | 升级动作 |
| --- | --- |
| 当前幕内未解决 | 在看板标注阻塞原因，通知直接上级 |
| 跨 1 幕仍未解决 | 升级至 GM，GM 决定调整方案或重新排期 |
| 跨 2 幕仍未解决 | GM 与甲方沟通，考虑放弃该 KR 或调整 Objective |

### 阻塞处理原则

- 阻塞不是失败，及时暴露比隐瞒更重要
- 阻塞原因应具体到可操作的下一步（"等待 X 提供 Y"而非"被阻塞了"）
- 解除阻塞后需更新看板并补充说明
